API Endpoints

The application provides REST API endpoints for managing courses, grades, and analytics:

GET /api/courses – Get all courses
POST /api/courses – Add a new course

GET /api/grades – Get all student grades
POST /api/grades – Add or update grades

GET /api/analytics/summary – Get overall performance summary
GET /api/analytics/course-performance – Get course-wise analytics

GET /api/health – Check API server status


Development

Adding New Features

1. Create new route files inside the routes directory
2. Register the routes in app.py using Flask Blueprints
3. Add corresponding templates in templates
4. Update styles in static/css/style.css
5. Test the API endpoints and UI thoroughly


Database / Data Changes

1. Update data logic inside the respective route file (courses.py, grades.py)
2. Restart the Flask server after changes
3. Test API responses using Postman or browser


Deployment

Production Setup

1. Set environment variables:
FLASK_ENV=production
SECRET_KEY=your-production-secret-key

2. Run the application using a production WSGI server:
pip install gunicorn
gunicorn -w 4 -b 0.0.0.0:5000 app:app
